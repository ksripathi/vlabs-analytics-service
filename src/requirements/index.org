* Introduction 
  This document illustrates the requirements to build =Analytics Service= for
  Virtual Labs

* Definitions 

  + Usage ::

    - A user performing an experiments of a lab is considered to be a single
      usage. This is captured by click event on the simulation link of an
      experiment section of Open edX.

    - If there are 3 experiments in a lab, A lab will have 3 usages
      and experiment will have 1 usage.

* Requirements
  :PROPERTIES:
  :CUSTOM_ID: requirements
  :END: 
** Req1
  :PROPERTIES:
  :CUSTOM_ID: req1
  :END: 
  - Ability to design REST APIs to store/retrieve the usage analytics of the Virtual
    Labs hosted on Openedx, College Cloud and other platforms
** Req2 
  :PROPERTIES:
  :CUSTOM_ID: req2
  :END: 
  - Ability to design REST APIs to store/retrieve the user analytics of the
    Openedx platform running Virtual Labs
     
** Req3
  :PROPERTIES:
  :CUSTOM_ID: req3
  :END:
  - Ability to design REST APIs to store/retrieve the user analytics of the
    Outreach Portal

** Req4
  :PROPERTIES:
  :CUSTOM_ID: req4
  :END: 
  - Ability to design REST APIs to store/retrieve the feedback analytics of
    Virtual Labs hosted on Openedx, College Cloud and other platforms
